# Project Context

## Purpose
SenseFlow is a high-efficiency clipboard manager for macOS that automatically captures clipboard history (text and images), provides quick access via global hotkey, and supports features like auto-paste, OCR search, and customizable shortcuts. The goal is to enhance productivity by making clipboard history easily accessible and searchable.

## Tech Stack
- **UI Layer**: SwiftUI (content) + AppKit (window management)
- **Data Layer**: SQLite.swift for database management
- **System Integration**:
  - NSPasteboard API for clipboard monitoring
  - Carbon EventHotKey API for global shortcuts
  - CGEvent for auto-paste functionality
  - Vision framework for OCR text recognition
- **Window Effects**: NSPanel + NSVisualEffectView/NSGlassEffectView (macOS 15+)
- **Platform**: macOS 12.0+ (Monterey and above)
- **Language**: Swift 5.7+
- **Build Tool**: Xcode 14.0+

## Project Conventions

### Architecture Patterns
- **MVVM Architecture**: SwiftUI recommended pattern
- **Single Responsibility Principle**: Each manager handles one concern

- **Separation of Concerns**:

  - Services/ - Background services (monitoring, caching, OCR)
  - Models/ - Data structures
  - Views/ - UI components

## Domain Context

### Clipboard Management
- **Monitoring**: 0.75s polling interval on NSPasteboard.changeCount
- **Deduplication**: SHA256 hash-based to prevent duplicates
- **History Limit**: Configurable 50-500 items (default 200), FIFO eviction
- **Large Files**: >512KB stored separately on filesystem, not in database
- **Sensitive Data Filtering**: Automatically filters password manager data (TransientType, ConcealedType, AutoGeneratedType)

### Window Behavior
- **Position**: Bottom center of screen, 20pt from bottom
- **Size**: 300pt height, auto-width (max screen width - 40pt)
- **Animations**:
  - Show: 0.45s spring animation with 30pt upward slide
  - Hide: 0.35s fade out with downward slide
- **Auto-hide**: Loses focus triggers hide
- **Material**: `.popover` glass effect (macOS 15+), falls back to `.regular`

### OCR Integration
- **Framework**: Vision framework (VNRecognizeTextRequest)
- **Languages**: Chinese (Simplified/Traditional) + English
- **Mode**: Accurate recognition with language correction
- **Processing**: Background async processing on image insert
- **Search**: OCR text indexed and searchable alongside regular text

## Important Constraints

### System Requirements
- macOS 12.0+ required (some features need 13.0+)
- Settings panel requires macOS 13.0+ (`.formStyle(.grouped)`)
- Liquid Glass effects require macOS 15.0+
- Accessibility permission required for auto-paste feature

### Debug Mode Limitations
- Debug builds require re-authorization of Accessibility permission after each compile
- This is due to code signature changes per build
- Release builds only need one-time authorization

### Performance Constraints
- Must maintain <0.1% CPU usage during monitoring
- Database operations must complete in <50ms
- UI must remain responsive at 60fps
- Memory usage must be reasonable with 200+ items

### Security Constraints
- Must filter sensitive data from password managers
- Must not capture system-generated temporary data
- Must pause monitoring when writing to clipboard (avoid self-capture)
- Large files must be stored securely outside database

## Important Development Rules

### API Modernization
- **CRITICAL**: Before implementing any new feature, query Context7 for latest API documentation
- Always use the most recent, recommended APIs
- Avoid deprecated or outdated APIs
- Example: When adding OCR, first research Vision framework latest APIs via Context7

### Xcode Project File Management
- When adding new files, manually update `SenseFlow.xcodeproj/project.pbxproj`
- Required sections: PBXBuildFile, PBXFileReference, PBXGroup, PBXSourcesBuildPhase
- Use unique IDs with prefixes (B/F/G + number)
- Maintain consistent formatting and structure
