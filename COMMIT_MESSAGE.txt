feat(arch): implement Clean Architecture

Refactor from singleton Manager pattern to Clean Architecture with
SOLID principles, dependency injection, and protocol-oriented design.

## Architecture Layers

- **Domain**: Protocols + Value Objects (9 files)
  - Protocols: ClipboardRepository, PromptToolRepository, HotKeyRegistry,
    ClipboardReader/Writer, AIServiceProtocol, NotificationServiceProtocol
  - Value Objects: ToolID, KeyCombo, ClipboardContent

- **UseCases**: Business logic (3 files)
  - ExecutePromptTool, AnalyzeAndRecommend, RegisterToolHotKey

- **Adapters**: Protocol implementations (6 files)
  - SQLitePromptToolRepository, OpenAIServiceAdapter, NSPasteboardAdapter,
    UserNotificationAdapter, CarbonHotKeyAdapter, SystemContextCollector

- **Infrastructure**: DI Container (1 file)
  - DependencyContainer

- **Coordinators**: Orchestration (2 files)
  - PromptToolCoordinator, SmartToolCoordinator

## Key Improvements

- ✅ Dependency Inversion: Depend on abstractions, not concrete classes
- ✅ Single Responsibility: Each class has one reason to change
- ✅ Testability: All dependencies can be mocked
- ✅ Type Safety: Value objects wrap primitive types (ToolID vs UUID)
- ✅ Loose Coupling: Protocol-oriented design throughout
