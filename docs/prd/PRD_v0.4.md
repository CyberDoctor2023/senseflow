# SenseFlow PRD v0.4 - Langfuse 云端同步

> **版本**: 0.4.0 | **平台**: macOS 26.0+ | **日期**: 2026-01-27

---

## 一、背景

### 1.1 当前已实现功能

- ✅ 划词工具基础功能（创建/编辑/删除/执行）
- ✅ 快捷键绑定与冲突检测
- ✅ AI 服务集成（OpenAI/Claude/DeepSeek/Ollama）
- ✅ Keychain 安全存储 API Key
- ✅ Smart 智能推荐（上下文感知自动推荐）

### 1.2 当前痛点

| 痛点         | 影响                    |
| ------------ | ----------------------- |
| 工具仅存本地 | 新工具必须等 App 更新   |
| 无版本控制   | 改错无法恢复，历史丢失  |
| 无法远程优化 | 运营无法实时调整 Prompt |
| 多设备不同步 | 换电脑要重新配置        |

### 1.3 目标用户

| 角色     | 诉求                   |
| -------- | ---------------------- |
| 普通用户 | 自动更新，不用操心     |
| 高级用户 | 自建工具不受影响       |
| 运营人员 | 后台随时发布/更新/回滚 |

## 二、功能目标

将划词工具从本地存储迁移到 Langfuse 云端管理：

| 目标         | 描述                                  |
| ------------ | ------------------------------------- |
| **远程管理** | 云端发布工具，无需更新 App            |
| **版本控制** | 支持版本号和标签，可回滚              |
| **自动同步** | 后台定时同步（默认5分钟），用户无感知 |
| **离线支持** | 断网时使用本地缓存，正常工作          |
| **向后兼容** | 用户自建的本地工具不受影响            |

### 核心原则

- **云端工具**：只读，由运营在 Langfuse 后台管理
- **本地工具**：用户自建，完全独立，不参与同步

---

## 三、同步机制

### 3.1 单向同步

采用「**云端检索，本地更新**」模式：

| 方向        | 支持 | 说明                               |
| ----------- | ---- | ---------------------------------- |
| 云端 → 本地 | ✅   | 自动从 Langfuse 拉取，覆盖本地缓存 |
| 本地 → 云端 | ❌   | 不支持推送，需在 Langfuse 后台编辑 |

### 3.2 三层缓存

工具执行时按顺序查找：

| 层级          | 响应速度 | 有效期 | 用途                         |
| ------------- | -------- | ------ | ---------------------------- |
| 内存缓存      | 最快     | 60秒   | 快捷键连续触发时极速响应     |
| 本地数据库    | 快       | 永久   | 离线时的数据来源             |
| Langfuse 云端 | 较慢     | -      | 权威数据源，缓存没命中时请求 |

### 3.3 定时同步流程

1. 每隔设定时间（默认5分钟）检查云端
2. 比对「更新时间」，只下载有变化的（增量更新）
3. 更新本地数据库，清除内存缓存
4. 工具列表自动刷新

### 3.4 冲突处理（远程优先）

| 云端变化 | 本地处理       |
| -------- | -------------- |
| 新增工具 | 添加到本地缓存 |
| 更新工具 | 覆盖本地缓存   |
| 删除工具 | 从本地缓存移除 |

> 💡 用户自建的本地工具永远不会被覆盖，两类工具完全独立。

### 3.5 离线机制

| 网络状态 | 行为                   |
| -------- | ---------------------- |
| 正常联网 | 定时同步云端           |
| 断网     | 使用本地缓存，功能正常 |
| 网络恢复 | 自动重新同步           |

## 五、数据需求

### 5.1 工具来源标识

通过 `source` 字段区分工具来源，**云端同步只影响 langfuse 来源的工具**：

| 来源值     | 含义             | 是否参与同步          |
| ---------- | ---------------- | --------------------- |
| `custom`   | 用户自建         | ❌ 永不同步，完全本地 |
| `langfuse` | 云端下发         | ✅ 自动同步更新       |
| `builtin`  | 旧版内置（历史） | ❌ 保持不变           |

> 💡 **关键逻辑**：同步时根据 `source` 判断，`custom` 来源的工具永远不会被覆盖或删除。

**用户编辑云端工具的处理**：

当用户尝试编辑 `langfuse` 来源的工具时，采用「**复制后编辑**」策略：

1. 用户点击编辑 → 提示「将创建本地副本」
2. 自动复制一份，`source` 改为 `custom`
3. 原云端工具保持不变，继续参与同步
4. 副本成为独立的本地工具，用户可自由修改

### 5.2 云端工具扩展字段

Langfuse 来源的工具需要额外记录：

| 字段            | 说明                             |
| --------------- | -------------------------------- |
| langfuseName    | 云端工具唯一标识（用于匹配更新） |
| langfuseVersion | 当前版本号                       |
| langfuseLabel   | 标签（production/staging）       |
| lastSyncedAt    | 最后同步时间                     |

### 5.3 配置数据

| 配置项   | 存储位置   |
| -------- | ---------- |
| 启用状态 | 用户偏好   |
| 同步间隔 | 用户偏好   |
| API 密钥 | 系统钥匙串 |

## 七、边界情况

| 场景               | 处理                 |
| ------------------ | -------------------- |
| 云端与本地工具同名 | 独立存在，不冲突     |
| 云端工具被删除     | 同步后从本地移除     |
| 同步中断           | 保留上次成功数据     |
| 密钥过期           | 提示重配，继续用缓存 |
| 首次同步 50+ 工具  | 显示同步进度         |
