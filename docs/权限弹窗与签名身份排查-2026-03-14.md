# 权限弹窗与签名身份排查结论（2026-03-14）

## 一、核心结论

1. 通知权限“点了不弹窗”与代码逻辑、系统一次性授权机制、以及构建签名身份不一致三者叠加有关。
2. GitHub Release 当前产物是“显式关闭代码签名”的构建，系统会将其识别为与本地已授权版本不同的应用身份。
3. 当应用身份变化（即使 Bundle ID 相同）时，TCC/系统隐私权限会按“新应用”处理，需要重新授权，系统设置中会出现旧版本与新版本并存条目。

## 二、代码与配置证据

### 1) GitHub Release 关闭签名（高优先级）

文件：`.github/workflows/release.yml`

- 构建参数包含：
  - `CODE_SIGN_IDENTITY="-"`
  - `CODE_SIGNING_REQUIRED=NO`
  - `CODE_SIGNING_ALLOWED=NO`

这会导致发布包身份与本地已签名版本不一致，直接触发权限重置现象。

### 2) 项目本身 Bundle ID 与 Team 是固定的

文件：`SenseFlow.xcodeproj/project.pbxproj`

- `PRODUCT_BUNDLE_IDENTIFIER = com.senseflow.SenseFlow`
- `DEVELOPMENT_TEAM = FPVWZ792P2`

说明“身份漂移”的主要来源不是工程配置本身，而是 CI 发布流程禁签。

### 3) 通知权限请求逻辑（系统机制）

文件：`SenseFlow/Services/NotificationService.swift`

- 仅在 `authorizationStatus == .notDetermined` 时触发系统授权请求。
- 若状态是 `.denied`，系统不会再次弹出授权框，只能引导用户去系统设置开启。

## 三、权限状态是否实时读取

### 1) 引导页（Onboarding）

文件：`SenseFlow/Views/OnboardingView.swift`

- 有定时器轮询（`BusinessRules.Permissions.checkInterval`，当前为 0.5s），属于实时刷新权限状态。

### 2) 设置页（Privacy）

文件：`SenseFlow/Views/Settings/PrivacySettingsView.swift`

- 当前是在 `onAppear` 时读取一次，并非持续轮询。
- 因此设置页可能出现“状态滞后”，尤其在用户刚切到系统设置修改权限后。

## 四、与 Apple 文档一致的行为

1. `UNUserNotificationCenter.requestAuthorization` 仅在首次请求（notDetermined）时触发系统授权弹窗。
2. 用户已拒绝后，系统会记住选择，不会反复弹窗；需去系统设置手动开启，或测试时重置权限。

参考：
- https://developer.apple.com/documentation/usernotifications/asking-permission-to-use-notifications
- https://developer.apple.com/documentation/uikit/requesting-access-to-protected-resources

## 五、下一步修复建议（执行优先级）

1. 将 GitHub Release 改为稳定签名链路（Developer ID + 同一 Team），避免每次下载后权限被当成新应用重置。
2. 设置页权限状态改为实时刷新（定时轮询或前后台切换触发刷新），与引导页一致。
3. 通知权限按钮点击后，明确提示当前系统状态（notDetermined/denied/authorized），减少误判。

## 六、轮询与性能的架构取舍（调研结论）

1. **最佳实践不是“全局高频轮询”**，而是“事件驱动为主，轮询仅限短交互期”。  
2. SwiftUI 推荐通过 `scenePhase` / 应用激活事件触发状态刷新，适合设置页这类长期停留页面。  
3. `Timer` 若必须使用，应设置 `tolerance`（Apple 建议重复定时器至少 10% 容差）以降低能耗。  
4. 对本项目的建议：  
   - Onboarding：允许 0.5s 短期轮询（用户正在授权交互）。  
   - Settings：不做常驻高频轮询，采用 `didBecomeActive` 事件刷新。  

参考：  
- https://developer.apple.com/documentation/swiftui/scenephase  
- https://developer.apple.com/documentation/foundation/timer
