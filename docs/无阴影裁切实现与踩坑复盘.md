# 无阴影裁切实现与踩坑复盘

## 1. 目标与结论

目标：让顶部搜索区和卡片区的 `glassEffect` 阴影完整显示，不被窗口边界裁切，同时不改动你已有的搜索栏动画接口与动画参数。

最终结论：这个问题不是“单点参数”问题，而是 **多层渲染边界叠加** 问题。必须同时满足以下条件才稳定：

1. `NSHostingView` 不裁切内容。
2. 顶层 SwiftUI 布局不给 glass 强行定高裁切。
3. `NSPanel` 不使用系统阴影（避免和 glass 阴影打架）。
4. 窗口尺寸与定位要为阴影留 bleed 空间（内容留白 + 窗口 frame 补偿）。

---

## 2. 现象与根因模型

### 2.1 现象

- 搜索栏或卡片边缘阴影被切掉，看起来像“贴着边界断掉”。
- 某些版本会出现“有时正常、有时又裁切”的不稳定表现。

### 2.2 根因模型（按层）

从外到内看渲染链路：

1. `NSPanel`（窗口外框）
2. `NSHostingView`（SwiftUI 宿主）
3. SwiftUI 根容器（`UnifiedPanelView`）
4. 具体 glass 视图（`EmptyBackgroundView` / `ClipboardListView`）

只要任意一层把可视区域卡死，阴影就会被切。

## 4. 最终实现（当前代码）

## 4.1 宿主层不裁切

文件：`SenseFlow/Managers/Window/WindowFactory.swift:69`

```swift
hostingView.clipsToBounds = false
```

作用：允许 SwiftUI glass 阴影超出 hostingView 边界渲染。

## 4.2 系统阴影关闭

文件：`SenseFlow/Managers/Window/WindowConfigurator.swift:41`

```swift
panel.hasShadow = false
```

作用：只保留 glass 自己的阴影，避免双阴影叠加锯齿/错位。

## 4.3 顶部视图不做硬裁切

文件：`SenseFlow/Views/UnifiedPanelView.swift:39`
文件：`SenseFlow/Views/EmptyBackgroundView.swift:94`
文件：`SenseFlow/Views/EmptyBackgroundView.swift:126`

关键点：

- `UnifiedPanelView` 用 `ZStack` 的透明占位，不对 `EmptyBackgroundView` 本体强行 frame 裁切。
- `EmptyBackgroundView` 的搜索胶囊移除固定高度 `frame(height:)`，让 glass 自主决定最终外扩。

## 4.4 恢复窗口级 shadow bleed 机制（本次回补）

### 常量定义

文件：`SenseFlow/Styles/Constants.swift:24`

- `shadowBleedHorizontal = 10`
- `shadowBleedTop = 10`
- `shadowBleedBottom = 10`

### 统一窗口尺寸计算

文件：`SenseFlow/Protocols/WindowLayoutConfigurable.swift:50`

- `unifiedWindowHeight = 内容高度 + verticalBleed`
- `unifiedWindowWidth = 屏幕宽度扣 inset 后再加 horizontalBleed`

### 窗口 frame 补偿

文件：`SenseFlow/Managers/Window/WindowPositioner.swift:34`

- 宽度使用 `unifiedWindowWidth(...)`
- `x/y` 分别减去对应 bleed 偏移，把窗口原点外移，给阴影腾位置

### 根视图内容留白

文件：`SenseFlow/Views/UnifiedPanelView.swift:61`

- 对根容器增加 `horizontal/top/bottom` 的 shadow bleed padding

## 5. 为什么以前会“改了又复发”

常见误区：

1. 只改一层（比如只改 `clipsToBounds`）
   其它层仍可能裁切，问题会在不同场景复发。

2. 用 `panel.hasShadow = true` 试图补偿
   表面可能“看起来好了”，但会引入双阴影冲突，不是稳定方案。

3. 删除/缺失 bleed 机制
   当内容贴近窗口边缘时，阴影再次触边被切。

4. 在 glass 前面加硬 `frame(height:)`
   等于直接限制了 glass 的外扩空间。

## 6. 回归检查清单（每次改动后必做）

1. 检查 `NSHostingView.clipsToBounds == false`。
2. 检查 `panel.hasShadow == false`。
3. 检查搜索胶囊是否重新出现 `.frame(height:)`（尤其在 `.glassEffect` 前后）。
4. 检查 `UnifiedPanelView` 是否仍保留 shadow bleed padding。
5. 检查 `WindowPositioner` 是否仍在使用 `unifiedWindowWidth` 与 `x/y` bleed 偏移。
6. 多屏切换、失焦/回焦、首次弹窗三种场景都手动验证一次。
