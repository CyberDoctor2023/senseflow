# 2026-02-06 改进记录

## 改进概述

基于 Maccy 和 Deck 的源码分析，实现了三个关键改进：

1. **改用"忽略下一次"机制**（替代时间暂停）
2. **支持文件类型**（.fileURL）
3. **重置 ScrollView 滚动位置**（每次打开回到最左边）

---

## 改进 #1: 忽略下一次变化

### 问题

**旧方案**：时间暂停（有缺陷）
```swift
// 暂停 1 秒
ClipboardMonitor.shared.pauseMonitoring(duration: 1.0)
NSPasteboard.general.setString(selectedText)
```

**缺陷**：
- ❌ 暂停期内的**所有**剪贴板变化都被忽略
- ❌ 可能误伤用户的快速操作（0.3s 内复制文件会被忽略）
- ❌ 时间窗口不精确（系统慢时可能失效）

### 解决方案

**新方案**：忽略下一次（精确）
```swift
// 设置标志位
ClipboardMonitor.shared.ignoreNextChange()
NSPasteboard.general.setString(selectedText)

// 监听器
func checkPasteboard() {
    if shouldIgnoreNextChange {
        shouldIgnoreNextChange = false  // 重置标志
        return  // 只忽略这一次
    }
    // 正常处理
}
```

**优点**：
- ✅ 只忽略"下一次"变化（精确）
- ✅ 不依赖时间窗口
- ✅ 不会误伤用户的快速操作

### 代码变更

**ClipboardMonitor.swift**:
- 属性：`isPaused: Bool` → `shouldIgnoreNextChange: Bool`
- 方法：`pauseMonitoring(duration:)` → `ignoreNextChange()`（旧方法标记为 deprecated）

## 改进 #2: 支持文件类型

### 问题

**旧实现**：只检查图片和文本
```swift
if let imageData = getImageData() {
    handleImageData(...)
} else if let textContent = getTextContent() {
    handleTextContent(...)
} else {
    print("⚠️ 不支持的剪贴板类型")  // 文件被忽略了！
}
```

**为什么"有时能复制上"？**
- macOS 复制文件时会同时放入：
  - `.fileURL` 类型（文件引用）
  - `.string` 类型（文本路径 `/Users/jack/file.txt`）
- 我们捕获的是**文本路径**，不是真正的文件引用

### 解决方案

**学习 Maccy/Deck 的方式**：
1. 遍历所有 `pasteboardItems`
2. 支持 `.fileURL` 类型
3. 使用 `readObjects(forClasses: [NSURL.self])` 读取文件

```swift
// 新增方法
private func getFileURLs() -> [URL]? {
    // 方法 1: 使用 readObjects（推荐）
    if let urls = pasteboard.readObjects(forClasses: [NSURL.self]) as? [URL] {
        return urls
    }

    // 方法 2: 从 propertyList 读取（兼容）
    if let urlStrings = pasteboard.propertyList(forType: .fileURL) as? [String] {
        return urlStrings.compactMap { URL(string: $0) }
    }

    return nil
}

// 优先级检测
if let imageData = getImageData() {
    handleImageData(...)
} else if let fileURLs = getFileURLs() {
    handleFileURLs(...)  // 新增
} else if let textContent = getTextContent() {
    handleTextContent(...)
}
```

### 代码变更

**ClipboardMonitor.swift**:
- 新增方法：`getFileURLs() -> [URL]?`
- 新增方法：`handleFileURLs(_ urls: [URL], appInfo:)`
- 修改：`processPasteboardContent()` 增加文件类型检测

## 改进 #3: 重置 ScrollView 滚动位置

### 问题

**现象 1**：
- 用户上次滚动到右边
- 下次打开窗口，还在右边（看不到最新的项目）

**现象 2**（修复后发现）：
- 使用 `scrollTo(firstItem.id, anchor: .leading)` 后
- 第一个卡片紧贴左边缘，没有 padding
- 需要手动往右滑一下才到正确位置

**原因**：
- SwiftUI 的 `ScrollView` 会记住滚动偏移
- 使用 `.padding(.horizontal)` 在 LazyHStack 上时，`scrollTo()` 会忽略这个 padding

### 解决方案

**Apple 官方推荐方式**（使用 Spacer 占位符）：
```swift
ScrollViewReader { proxy in
    ScrollView(.horizontal) {
        LazyHStack {
            // 左边距占位符
            Spacer().frame(width: 16).id("leading-spacer")

            ForEach(viewModel.items) { item in
                ClipboardCardView(item: item)
                    .id(item.id)
            }

            // 右边距占位符
            Spacer().frame(width: 16)
        }
    }
    .onReceive(NotificationCenter.default.publisher(for: .windowWillShow)) { _ in
        // 滚动到左边距占位符（而不是第一个卡片）
        proxy.scrollTo("leading-spacer", anchor: .leading)
    }
}
```

**关键点**：
- 移除 LazyHStack 的 `.padding(.horizontal)`
- 在开头和结尾添加 `Spacer` 占位符
- 滚动到 `"leading-spacer"` 而不是第一个卡片
- 这样第一个卡片会有正确的 16pt 左边距

### 代码变更

**ClipboardListView.swift**:
- 移除 LazyHStack 的 `.padding(.horizontal)`
- 添加左右 Spacer 占位符（各 16pt）
- 滚动目标改为 `"leading-spacer"`
- 使用 `anchor: .leading` 确保对齐
