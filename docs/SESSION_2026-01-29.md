## 🐛 核心问题：Settings 窗口控制按钮异常

### 用户报告的问题
1. ❌ 只显示关闭按钮，没有最小化和最大化
2. ❌ 红绿灯按钮紧贴窗口顶部（无正常间距）
3. ❌ 侧边栏折叠按钮居中显示（应该在左侧）
4. ❌ 窗口无法调整大小

### 根本原因分析

经过系统性调研发现：**`Info.plist` 中的 `LSUIElement = true`** 是根本原因。

```xml
<key>LSUIElement</key>
<true/>
```

**LSUIElement 的影响**：
- 将应用标记为后台菜单栏应用（不在 Dock 显示）
- 限制所有窗口的控制按钮功能
- 导致标题栏布局异常

---

## 🔍 调研过程（5 轮 Apple 文档调研）

### 第 1 轮：基础概念
- Settings Scene vs UtilityWindow vs WindowGroup
- **发现**：UtilityWindow 用于辅助工具窗口，不适合设置面板

### 第 2 轮：布局方案对比
- TabView `.sidebarAdaptable` vs NavigationSplitView
- **发现**：两者都有标题栏问题，不是布局组件的问题

### 第 3 轮：窗口样式调研
- NSWindow.StyleMask 配置
- windowResizability 修饰符
- **发现**：纯 SwiftUI 修饰符无法解决 LSUIElement 限制

### 第 4 轮：社区反馈
- Reddit r/SwiftUI 讨论
- Stack Overflow 案例
- **发现**：LSUIElement 应用需要用 AppKit 直接配置窗口

### 第 5 轮：最终方案
- NSViewRepresentable 桥接
- 直接设置 NSWindow.styleMask
- **成功**：完全控制窗口样式

## ✅ 解决方案

### 方案 1：WindowConfigurator.swift（主要方案）

创建自定义 NSViewRepresentable：

```swift
struct WindowConfigurator: NSViewRepresentable {
    func makeNSView(context: Context) -> NSView {
        let view = NSView()
        DispatchQueue.main.async {
            guard let window = view.window else { return }

            // 设置完整的 styleMask
            window.styleMask = [
                .titled,           // 标题栏
                .closable,         // 关闭按钮
                .miniaturizable,   // 最小化按钮
                .resizable         // 最大化按钮 + 可调整大小
            ]

            // 修复标题栏外观
            window.titlebarAppearsTransparent = false
            window.titleVisibility = .visible
        }
        return view
    }
}
```

**集成到 SettingsView**：
```swift
.background(WindowConfigurator())
```

### 方案 2：AppDelegate 窗口监听（备用方案）

监听窗口创建通知，自动配置：

```swift
NotificationCenter.default.addObserver(
    forName: NSWindow.didBecomeKeyNotification,
    object: nil,
    queue: .main
) { notification in
    guard let window = notification.object as? NSWindow else { return }
    if window.title.contains("设置") {
        window.styleMask = [.titled, .closable, .miniaturizable, .resizable]
    }
}
```

### 方案 3：自定义 Sidebar 布局

放弃 TabView `.sidebarAdaptable`，使用完全自定义的布局：

```swift
HStack(spacing: 0) {
    SettingsSidebar(selectedSection: $selectedSection)
        .frame(width: 200)
    Divider()
    SettingsDetailView(selectedSection: selectedSection)
}
.background(WindowConfigurator())
```

**优势**：
- 完全控制布局，无折叠按钮
- 避免 TabView 的标题栏冲突
- 更简洁的代码
