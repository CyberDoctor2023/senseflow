# SenseFlow v0.5 测试指南

## 测试 1: 菜单栏动画图标

### 步骤
1. 在 Xcode 中运行应用（Cmd+R）
2. 观察菜单栏右上角图标（备忘录 + 3 个点）
3. 复制任意内容（Cmd+C）
4. 观察动画效果

### 预期结果
✅ 点点从左到右滑动
✅ 活跃的点变亮并放大
✅ 完成 2 个完整循环后停止
✅ 总时长约 900ms

## 测试 2: 划词复制（标准模式）

### 前置条件
- 打开设置 → 高级设置
- 确认"启用划词即复制"已开启
- 确认"启用强制取词"已关闭

### 步骤
1. 打开备忘录
2. 输入一些文本
3. 用鼠标选中文本（至少 3 个字符）
4. 释放鼠标

   ✅ 划词复制功能已启用
   ✅ 主提取器成功
   📝 获取到选中文本: xxx...
   ✂️ 划词复制: xxx...

✅ 按 Cmd+Option+V 打开历史窗口
✅ 刚才选中的文本出现在历史记录中
✅ 菜单栏图标有动画

### 支持的应用
✅ 备忘录
✅ TextEdit
✅ Safari
✅ Pages
⚠️ VSCode/Cursor/Xcode（需要强制取词）

## 测试 6: 设置实时切换

### 步骤
1. 打开设置 → 高级设置
2. 关闭"启用划词即复制"
3. 选中文本 → 不应复制
4. 开启"启用划词即复制"
5. 选中文本 → 应该复制

### 预期结果
✅ 设置立即生效，无需重启

## 测试 7: 强制取词开关切换

### 步骤
1. 在 VSCode 中选中文本（强制取词关闭）
2. 观察是否复制

### 预期结果

## 故障排查

### 问题 1: 划词复制不工作
- 检查"启用划词即复制"是否开启
- 检查 Accessibility 权限是否授予
- 查看控制台是否有错误日志

### 问题 2: Electron 应用不支持
- 开启"启用强制取词"
- 重新测试

### 问题 3: 菜单栏图标无动画
- 检查是否真的保存到了数据库
- 查看控制台是否有"📝 保存文本"日志

### 问题 4: 重复保存
- 检查是否有多个应用实例在运行
- 查看暂停/恢复监听的日志
