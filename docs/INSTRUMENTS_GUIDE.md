# Instruments 性能诊断指南

## 目标

诊断窗口打开时的闪烁问题，找到性能瓶颈。

## 测试场景

1. 按快捷键 (⌘⇧V) 打开窗口
2. 观察窗口滑入动画 (0-350ms)
3. 观察列表渲染和图片加载

## 使用 Core Animation 模板

### 启动方式
```bash
# 方式 1：命令行
open -a Instruments

# 方式 2：Xcode
# Product → Profile (⌘I)
```

### 关键指标

#### 1. Frame Rate (帧率)
- **目标**: 60 fps
- **警告**: < 50 fps
- **严重**: < 30 fps
- **检查**: 窗口动画期间 (0-350ms) 的帧率曲线

#### 2. GPU Utilization (GPU 使用率)
- **正常**: < 50%
- **警告**: 50-80%
- **严重**: > 80%
- **检查**: 是否有突然的峰值

#### 3. Offscreen Rendering (离屏渲染)
- **检查**: `.drawingGroup()` 是否导致过多离屏渲染
- **位置**: Core Animation → Debug Options → Color Offscreen-Rendered Yellow

#### 4. View Body Duration (视图渲染时间)
- **目标**: < 16ms (60fps)
- **警告**: 16-33ms (30-60fps)
- **严重**: > 33ms (< 30fps)

### 具体检查点

#### 检查点 1：窗口动画阶段 (0-350ms)
- [ ] 帧率是否稳定在 60fps
- [ ] GPU 使用率是否正常
- [ ] 是否有掉帧（Frame Drop）

#### 检查点 2：列表首次渲染
- [ ] LazyHStack 渲染时间
- [ ] 是否有大量视图同时创建
- [ ] View Body Duration 是否超过 16ms

#### 检查点 3：图片加载
- [ ] 图片加载是否阻塞主线程
- [ ] 是否有同步 I/O 操作
- [ ] 图片解码时间

#### 检查点 4：.drawingGroup() 影响
- [ ] 离屏渲染次数
- [ ] GPU 合成时间
- [ ] 是否有不必要的离屏渲染

## 使用 SwiftUI Instrument (Xcode 17+)

### 关键指标

#### 1. View Updates
- **检查**: 哪些视图在频繁更新
- **问题**: 不必要的重新渲染

#### 2. State Changes
- **检查**: 状态变化的频率和时机
- **问题**: 过多的状态变化触发重新渲染

#### 3. Layout Calculations
- **检查**: 布局计算的时间
- **问题**: 复杂的布局导致性能下降

## 预期发现

基于调研，可能的问题：

### 问题 1：图片同步加载
- **现象**: 主线程阻塞，帧率下降
- **位置**: ClipboardCardView.swift:95 `item.getImage()`
- **影响**: 首次渲染时多个图片同时加载

### 问题 2：.drawingGroup() 开销
- **现象**: GPU 使用率峰值，离屏渲染
- **位置**: ClipboardCardView.swift:64 `.drawingGroup()`
- **影响**: 每个卡片都创建离屏图像

### 问题 3：数据加载时机
- **现象**: 视图先显示空状态，然后突然更新
- **位置**: ClipboardListView.swift:125 `.task`
- **影响**: 用户看到空状态→列表的闪烁

### 问题 4：动画时长
- **现象**: 动画时间过长，感觉迟钝
- **位置**: FloatingWindowAnimator.swift 350ms
- **影响**: 用户感觉应用响应慢
